# Backlog management

- Use checkboxes (`- [ ]` / `- [x]`) for all items.
- **Never prefix items with numbers** — it breaks markdown rendering.
- Organize by date/version, not by priority.
- Mark completed items with `[x]`.
- Implement sequentially: commit each item, then move to next.
- When displaying the backlog, hide completed items and number
  remaining items sequentially for easy selection.
- Display items in a table with columns: #, Item, Impact, Effort
  (use HIGH/MEDIUM/LOW). Skip the table when there are too many
  items (> ~15).
