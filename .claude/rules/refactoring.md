# Code refactoring

## When to suggest

- Every 6-7 versions.
- During long sessions with multiple files changed.
- When spotting code smells.

## Code smells

- Duplicated code/logic across modules.
- Large functions (delegate to helpers).
- Inconsistent naming/patterns.
- Dead code/unused imports.
- Overly complex conditionals.

## Output

Prioritized list (consistency first), added to `backlog.md`.
